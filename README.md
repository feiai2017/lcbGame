## 服务简介

利用自定义格式的TCP协议跟前端交互，socket断开连接时候保存数据到MongoDB

| 目录结构 | 说明 |
| :--- | :--- |
| / | 项目根目录 |
| /config | 配置文件目录 |
| service | 服务端代码目录 |
|internal | fakeclient|
| log | 日志目录 |

## 逻辑简述
用户通过自定义协议跟服务端交互，服务端接收到开始游戏请求后, 会根据当前人数初始化棋盘和序列，游戏结束后，服务端会将游戏数据保存到MongoDB中

## 响应结果

**成功响应**

当 code 状态码为 ``00000`` 时，表明调用成功。
